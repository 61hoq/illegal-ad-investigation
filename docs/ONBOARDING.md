# 온보딩 가이드 — 방치 사이트 불법광고 추적 프로젝트

새로 합류한 팀원을 위한 안내 문서입니다. 이 문서 하나로 **프로젝트가 뭔지, 지금까지 뭘 했는지, 앞으로 뭘 할지, 어디서부터 시작하면 되는지**를 파악할 수 있도록 정리했습니다.

---

## 1. 한눈에 보기

관리가 중단된 **방치 웹사이트의 게시판**이 마약·도박·불법대출 같은 범죄 광고의 유통 창구로 쓰이고 있습니다. 우리는 이런 사이트 65곳에서 게시글 **416,006건**을 크롤링해, 어떤 범죄가 얼마나 성행하는지, 어떤 텔레그램 아이디가 어느 사이트를 공급망으로 쓰는지를 데이터로 분석했습니다.

- 규모: 65개 사이트 · 41.6만 건 · 17개 범죄 카테고리 · 825개 키워드 사전
- 핵심 결과: 약물 게시글이 마약 유통의 우회로였고, 소수 텔레그램 아이디가 유통을 주도. 마약 홍보는 수도권·경남·경북·강원에 집중.
- 목표: 방치 사이트가 범죄 통로가 된다는 사실을 데이터로 드러내고, 법제화·단속·제재의 필요성을 제기하는 데이터 저널리즘.

## 2. 왜 하는가

불법 광고 게시글이 올라와도 관리자가 없어 방치되는 사이트들이 있습니다. 이런 게시판은 검색에 노출되며 범죄 거래의 접점이 됩니다. 그러나 이를 규제할 법적 근거는 아직 공백에 가깝습니다. 우리는 "얼마나, 어디서, 어떻게" 벌어지는지를 정량적으로 보여줌으로써 이 사각지대를 공론화하려 합니다.

## 3. 지금까지 한 일 (파이프라인)

```
사이트 수집 → 크롤링 → 전처리 → 분류 → 지명 매핑 → 분석·시각화
 (65개)     (41.6만건)  (특수문자)  (규칙+NB   (법정동코드)  (공급수요·생키·
                                  +트랜스포머)              지도·워드클라우드)
```

1. **데이터 수집**: 불법 광고 최초 사이트를 추적해 100여 개를 확보하고, 2026년에도 게시물이 올라오는 65개를 최종 선정. 매크로 과다 사이트는 최대 1만 건 제한(`is_truncated`). 컬럼을 통일해 병합, URL 기준 중복 제거. 최종 416,006건.
2. **전처리**: 검색 회피용 특수문자(첨자·딩벳)를 NFKC 정규화로 복원, 러시아어 게시글 제거 → 415,550건.
3. **사전 구축**: 불법 유형 17개 카테고리에 825개 키워드 사전 + 법정동코드 기반 지명 사전(19,832행).
4. **형태소 분석**: Kiwi 사용자 사전에 키워드 등록 후 명사 추출(`nouns`).
5. **카테고리 분류**: ① 규칙 + 나이브 베이즈로 1차 분류(신뢰도 임계값 적용, 애매하면 미분류) → ② 신뢰도 높은 데이터로 **KLUE-RoBERTa** 파인튜닝(KoELECTRA도 비교 실험).
6. **지명 매핑**: 명사에서 지역명 추출해 `region_list` 생성(혼동 단어 예외 처리, 수도권 우선).
7. **분석·시각화**: 범죄 공급/수요, 텔레그램 아이디–범죄–사이트 생키 다이어그램, 지역별 마약 지도, 마약 종류·지역 워드클라우드 등.

> 시각화 예시 이미지는 `docs/figures/`에 있습니다. (공급수요·마약지도·워드클라우드)

## 4. 최근 추가로 밝혀낸 것

지난 제출 이후 추가로 확인한 내용입니다. 이어서 발전시킬 지점들입니다.

1. **SEO 경로 발견**: 방치 사이트 게시판뿐 아니라, 검색엔진(SEO)을 통해서도 같은 불법 광고에 도달할 수 있음을 확인. 즉 유입 경로가 게시판 하나가 아님.
2. **구글 트렌드 — 시간대 집중**: 관련 검색이 특정 시간대에 몰리는 패턴을 발견. 수요의 시간적 리듬이 존재함.
3. **신고 후 차단 여부 추적**: 마약·도박·해킹·불법대출·문서위조 카테고리를 신고하고, 실제로 사이트가 막히는지 관찰 중. (신고일 — 마약: 3/17, 나머지: 3/23) → 신고의 실효성 자체가 하나의 데이터가 됨.
4. **분류 보완**: 규칙+나이브베이즈 분류를 딥러닝(트랜스포머) 모델로 보완해 정확도 개선.

## 9. 반드시 지킬 것 (윤리 · 보안)

이 프로젝트는 범죄를 **드러내는** 것이 목적입니다. 다루는 자료의 성격상 아래를 꼭 지켜주세요.

- **원본 취재 자료는 공개 금지**: 마약 메뉴판·가격, 텔레그램 거래 대화, 신분증 위조·안락사 관련 캡처 등은 공개 저장소·발표 자료에 원본 그대로 싣지 않습니다.
- **텔레그램 아이디 익명화**: 생키 다이어그램 등에서 실제 판매자 아이디가 보이면, 공개용에서는 `ID-A`, `ID-B`처럼 가린 뒤 사용합니다. (실제 활성 아이디를 그대로 공개하면 범죄 조력 우려 + 판매자에게 노출됩니다.)
- **대용량·민감 데이터는 저장소에 커밋 금지**: 239MB 전처리 데이터는 `.gitignore`로 제외. 필요한 사람에겐 팀 내부 드라이브로 공유.
- **분석 목적 유지**: 어떤 산출물도 거래 방법을 안내하지 않도록 합니다. 우리는 구조를 보여주고 제도 개선을 제언하는 쪽입니다.
